# Fantasy Power Rankings

A weekly power rankings voting app for your Sleeper fantasy football leagues. Built for GitHub Pages — no server required.

## Features

- 🗳 **Weekly voting** — league members submit blind rankings each week
- 📊 **Results** — aggregate power rankings visible to all; individual ballots visible to commissioner only
- 📈 **History** — line graph tracking each team's ranking week over week
- 🤖 **AI Summary** — weekly league recap generated by the commissioner each Tuesday, powered by Claude

---

## Setup (takes about 10 minutes)

### Step 1 — Fork / upload to GitHub Pages

1. Create a new GitHub repository (e.g. `fantasy-power-rankings`)
2. Upload `index.html` to the root of the repo
3. Go to **Settings → Pages → Source** and set it to `main` branch, `/ (root)`
4. Your app will be live at `https://yourusername.github.io/fantasy-power-rankings`

---

### Step 2 — Create a Supabase project

Supabase is the free database that stores votes and summaries across all devices.

1. Go to [supabase.com](https://supabase.com) and sign up (free)
2. Click **New project**, give it a name like `power-rankings`, choose a region close to you, set a database password (save it somewhere)
3. Wait ~2 minutes for it to provision

---

### Step 3 — Create the database tables

In your Supabase project, go to **SQL Editor** and run each of these blocks:

#### Table 1: votes
```sql
create table votes (
  id uuid default gen_random_uuid() primary key,
  league_id text not null,
  week integer not null,
  voter_user_id text not null,
  voter_username text,
  voter_display_name text,
  rankings jsonb not null,
  submitted_at timestamptz default now(),
  unique (league_id, week, voter_user_id)
);

alter table votes enable row level security;
create policy "Anyone can read votes" on votes for select using (true);
create policy "Anyone can insert votes" on votes for insert with check (true);
create policy "Anyone can update own vote" on votes for update using (true);
```

#### Table 2: ranking_history
```sql
create table ranking_history (
  id uuid default gen_random_uuid() primary key,
  league_id text not null,
  week integer not null,
  roster_id text not null,
  team_name text,
  owner_name text,
  rank integer not null,
  avg_score numeric,
  unique (league_id, week, roster_id)
);

alter table ranking_history enable row level security;
create policy "Anyone can read history" on ranking_history for select using (true);
create policy "Anyone can insert history" on ranking_history for insert with check (true);
create policy "Anyone can update history" on ranking_history for update using (true);
```

#### Table 3: weekly_summaries
```sql
create table weekly_summaries (
  id uuid default gen_random_uuid() primary key,
  league_id text not null,
  week integer not null,
  summary_text text not null,
  generated_at timestamptz default now(),
  generated_by text,
  unique (league_id, week)
);

alter table weekly_summaries enable row level security;
create policy "Anyone can read summaries" on weekly_summaries for select using (true);
create policy "Anyone can insert summaries" on weekly_summaries for insert with check (true);
create policy "Anyone can update summaries" on weekly_summaries for update using (true);
```

---

### Step 4 — Get your Supabase credentials

1. In your Supabase project, go to **Settings → API**
2. Copy the **Project URL** (looks like `https://xxxxxxxxxxxx.supabase.co`)
3. Copy the **anon / public** key (the long string under "Project API keys")

---

### Step 5 — Add credentials to the app

Open `index.html` and find this section near the top of the `<script>` block:

```js
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```

Replace the placeholder values with your actual URL and anon key.

**Also update the commissioner password:**
```js
const COMMISSIONER_PASSWORD = 'PLACEHOLDER';
```
Change `PLACEHOLDER` to something only you'll know before pushing to GitHub.

Save the file and push/upload it to GitHub. Your site will update within a minute.

---

### Step 6 — Share with your league

Send everyone your GitHub Pages URL (e.g. `https://yourusername.github.io/fantasy-power-rankings`).

They log in with their Sleeper username. The app automatically detects which league(s) they're in from the two hardcoded league IDs.

## Commissioner Workflow

Each Tuesday during the regular season:

1. Visit the app and log in as `Purple_Kirsch`
2. Enter your commissioner password
3. Go to the **Summary** tab
4. Enter your Anthropic API key (get one free at [console.anthropic.com](https://console.anthropic.com))
5. Click **Generate Weekly Summary**

The summary is saved to Supabase and immediately visible to everyone in the league. Your API key is only used in your browser and is never stored.

> **Tip:** The Anthropic API key field is cleared each time you load the page. You only need to enter it when generating a new summary.

---

## Getting an Anthropic API Key

1. Go to [console.anthropic.com](https://console.anthropic.com)
2. Sign up / log in
3. Go to **API Keys** and create a new key
4. Generating one summary costs roughly $0.01–0.02 — essentially free

## Troubleshooting

**"Not in league" error on login**
Your Sleeper username might differ from your display name. In the Sleeper app: Profile → the @handle shown under your name (not your display name).

**Votes aren't showing up in Results**
Make sure all three SQL tables were created successfully. Go to Supabase → Table Editor and check that `votes`, `ranking_history`, and `weekly_summaries` all exist.

**AI summary failed**
Double-check your Anthropic API key. If it's your first time using the API, you may need to add a small credit (a few dollars lasts months at this usage level).

**Graph isn't showing**
The history graph only populates after the Results tab is loaded for the first time in a given week (that's what saves the snapshot). Make sure you visit Results after votes are in.
